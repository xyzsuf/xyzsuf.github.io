Dwm

November 26, 2022

a guide to configure dwm



dwm is a minimal dynamic window manager for X. The best part about dwm is that it does not have a config file, rather you need to configure it from source.

Prerequisites


Before installing dwm, we need a few packages just in case if it is not already installed.

sudo pacman -S vim git base-devel

Install

To install dwm, you can use git

git clone https://git.suckless.org/dwm



Additionally you would also need st –> suckless’ simple terminal, since dwm choose st as the default terminal emulator, and dmenu –> suckless’ dynamic menu.

git clone https://git.suckless.org/st && https://git.suckless.org/dmenu

Note: If you want to change the default MOD key to the windows key (instead of the ALT key) which is the default - you can cd into the dwm directory and open “config.h” file.

You need to change MOD1 to MOD4 as shown.

Also, if you would like to change the default keybinding to open up a terminal with (Winkey + return)

Replace the “ShiftMask” keybinding in line 56 as shown above, and add it in the zoom keybinding, or it will create a conflict while you compile the source code.

Once you have it installed, you need to cd into the dwm directory, where you had cloned the repository, and then type the following command.

 sudo make clean install

Follow the same steps for dmenu and st.
To launch dwm you need to set up ~/.xinitrc with atleast exec dwm

Adding DWM to Xsessions


If you are using a desktop environment, follow the commands below to add it to Xsessions.

cd /usr/share/xsesssions

Then, You can create a file by typing sudo vim DWM.desktop

And then, you can enter the following in the file verbatim:

[Desktop Entry]
Encoding=UTF-8
Name=DWM
Exec=/usr/local/bin/dwm
Type=Application

You can configure dwm by using the “config.h” file.

Adding Wallpaper


Install xwallpaper and add it to your ~/.xprofile.

Example:

xwallpaper --zoom ~/Pictures/image.jpeg
